## 1. אישור תחביר Claude Code

המימוש מבוסס על המפרט היציב של Claude Code לפקודות-פרויקט. **אימות דרך WebFetch/WebSearch נחסם בסשן הזה** — לכן הסתמכתי על הידע הקיים מהדוק של Claude Code:

- **מיקום:** פקודות-פרויקט חיות ב-`.claude/commands/*.md` בשורש הפרויקט (משותפות לכל מי שמשתמש ב-repo). פקודות-משתמש ב-`~/.claude/commands/` (אישיות).
- **שם הפקודה:** שם הקובץ בלי הסיומת. `daily.md` → `/daily`.
- **Namespacing (לא בשימוש כאן):** תתי-תיקיות הופכות לnamespace עם `:` — למשל `.claude/commands/rima/daily.md` → `/rima:daily`. בחרנו שלא, כדי להשאיר את השמות קצרים.
- **YAML frontmatter נתמך:**
  - `description` — תיאור קצר שמופיע ב-autocomplete.
  - `argument-hint` — טקסט עזר שמופיע כשהמשתמש מתחיל להקליד את הפקודה עם ארגומנטים.
  - `allowed-tools` — הגבלת סט הכלים שהפקודה יכולה להשתמש בהם.
  - `model` — (לא בשימוש כאן) בחירת מודל ספציפי לפקודה.
- **ארגומנטים:** `$ARGUMENTS` מחליף את הטקסט המלא שנכתב אחרי שם הפקודה. `$1`, `$2` וכו' לארגומנטים מיקומיים.
- **הפעלה:** CLI של Claude Code בלבד (אין גישה מה-API או ה-Web).

**דרך יחידה: Claude Code CLI.**

1. רימה פותחת טרמינל בתיקיית הפרויקט (`C:\Users\Avner\Documents\sipurima\`).
2. מריצה `claude` (או קיצור אחר שיוגדר לה בדסקטופ).

## 5. אילוצים ונקודות זהירות

### רינדור RTL בטרמינל
Windows Terminal ו-PowerShell מטפלים ב-RTL היטב ב-2025/2026, אבל cmd.exe הישן — פחות טוב. המלצה: **להעביר את רימה ל-Windows Terminal** (חינמי מה-Microsoft Store) כברירת מחדל.

### פונטים
טבלאות ASCII ואימוג'ים דורשים פונט מונוספייס עם כיסוי אימוג'ים. המלצה:
- **Cascadia Code** (ברירת המחדל של Windows Terminal — כבר מכיל את מה שצריך), או
- **JetBrains Mono** עם נפילה ל-Segoe UI Emoji.
לוודא ש-"Segoe UI Emoji" מותקן (הוא חלק מ-Windows).

### עברית ב-`$ARGUMENTS`
Claude Code תומך ב-UTF-8 ב-ארגומנטים. בדיקה פרקטית נדרשת בטרמינל של רימה ספציפית (יש אנקודינג שונה אם שפת ברירת המחדל של Windows היא עברית). אם יש בעיה — לוודא ש-`chcp 65001` רץ באוטומציה.

### עומס קריאה
חלק מהפקודות קוראות 3-4 קבצים. אם `STATUS.md` או `MEMORY.md` יגדלו מאוד (1000+ שורות), הפקודות יהיו איטיות יותר. לנטר ולשקול סיכום תקופתי.

## 6. שאלות פתוחות ל-QA

1. **רינדור אימוג'ים:** האם לאמת ידנית שהאימוג'ים (🔴 🟡 ✅ ⚪ 🔄 ⬜ ✨) נראים תקין בטרמינל של רימה? אם לא — יש להחליף לסימני ASCII (`[!]`, `[*]`, `[x]`, `[ ]`). אני מציע גרסה חלופית ASCII אם QA חושש.

2. **מקור היומן חסר:** `/daily` אמור להציג את פגישות היום. **לא מצאתי קובץ יומן** בפרויקט (`knowledge/calendar.md`? `knowledge/schedule.md`?). כרגע הפרומפט מורה ל-CEO לכתוב "אין מידע על היומן" אם אין מקור. החלטת מוצר: האם נוסיף קובץ יומן ב-v1 או דוחים ל-v2? אם v1 — לאיזה פורמט?

3. **אישור שפת-שמות אנגלית:** הסטרטג המליץ שמות אנגלית. מימשתי לפי ההמלצה. אם הקופירייטר / QA יחליט שהשמות צריכים להיות עבריים (`/יום`, `/משימות`) — הקבצים צריכים לעבור שינוי-שם. טכנית: שם קובץ עברי ב-`.claude/commands/יום.md` אפשרי, אבל טרם אומת בפועל מול Claude Code — יש סיכון תאימות.

4. **עקביות אימוג'י סטטוס:** הסטרטג הצביע על חוסר עקביות בין `STATUS.md` ל-`STRATEGY.md` (🔄 בעבודה או 🟡 בתהליך?). הפרומפטים שלי משתמשים ב-🔄 לעבודה פעילה ו-🟡 ל"תכנון/תהליך". צריך החלטה רשמית של הקופירייטר ועדכון תואם בקבצי המקור.

5. **האם `/decisions` צריך להציג גם החלטות שנהרסו/השתנו?** כרגע — רק "סגורות". אם ההחלטה הוחלפה, אין רישום. אולי QA רוצה להוסיף סעיף "החלטות שהתחלפו" לשקיפות?

6. **האם להוסיף namespace?** כרגע `/daily` גלובלי. אם נוסיף בעתיד פקודות למפתחים / לאבנר — שווה לשקול `/rima:daily`, `/dev:build` וכו'. לא צריך ב-v1.
