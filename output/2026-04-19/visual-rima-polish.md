### שדרוג של `blur_faces.py`

שכתבתי את `website/scripts/blur_faces.py` כדי לתמוך בשני מצבי-הגנה על רימה:

**מצב א' (מועדף): `face_recognition` (dlib).**
טוענים פעם אחת את ה-encoding של רימה מהפורטרט (`brand/from-rima/.../Screenshot_20240718_131943_WhatsApp.jpg`), ובכל תמונה משווים כל פרצוף שזוהה ומדלגים על זה שתואם לרימה.

**מצב ב' (fallback): Protect zones.**
אם `face_recognition` לא מותקן (dlib קשה לבנות על Windows), הסקריפט קורא `website/scripts/rima_zones.json` — קובץ שבו מגדירים לכל תמונת-מקור מלבן יחסי (0..1) שבתוכו רימה נמצאת. כל פרצוף שזוהה ושהמרכז שלו נופל בתוך המלבן — לא מטושטש.

הגדרתי ידנית את ה-zones על סמך הסתכלות ב-5 התמונות. שני המצבים יכולים לרוץ יחד (ה-zone מגבה את ה-identity).

### פרמטרים חדשים ל-CLI
```
python blur_faces.py <in_dir> <out_dir> \
  --rima-portrait "<path to Rima portrait jpg>" \
  --zones website/scripts/rima_zones.json \
  --pixel-size 20 --margin 0.25
```

`run_pipeline.bat` עודכן להשתמש בשני הדגלים.

### מה צריך לרוץ (לא רץ במסגרת הסשן הזה)
```bat
REM התקנה חד-פעמית (אם רוצים את מצב א')
C:\Users\Avner\AppData\Local\Programs\Python\Python312\python.exe -m pip install face_recognition

REM טשטוש מחדש (ה-pipeline כבר מגדיר הכל)
cd C:\Users\Avner\Documents\sipurima\website\scripts
run_pipeline.bat
```

אם face_recognition/dlib נופל בהתקנה — הסקריפט ייפול בעדינות חזרה ל-zone mode בלבד, שגם הוא מספיק כי הגדרתי ידנית את המיקום של רימה בכל תמונה. **אל תסגור את ה-zone file.**

לאחר ה-run, `make_responsive.py` ייצור את הגרסאות ב-480/768/1200 ב-webp+jpg. ה-HTML כבר מפנה לשמות הקבצים הנכונים — לא צריך לגעת.
